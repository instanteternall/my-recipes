# 部署方案参考文档

> 整理了几种适合国内用户的免费/低价部署方案，供后续选择使用。

---

## 方案一：Vercel（前端）+ Zeabur（后端）⭐ 推荐

**适合人群**：愿意付少量费用，追求稳定和国内访问速度

| 服务 | 平台 | 费用 | 说明 |
|------|------|------|------|
| 前端 | [Vercel](https://vercel.com) | 免费 | 自动 HTTPS，全球 CDN |
| 后端 | [Zeabur](https://zeabur.com) | 按用量计费，约 ¥15-30/月 | 香港节点，国内访问快 |
| 数据库 | Zeabur MySQL | 包含在后端费用内 | 一键部署 |

**优点**：
- Zeabur 总部香港，国内访问流畅
- 支持从 GitHub 一键导入部署
- 中文界面，操作简单

**部署配置文件**（已准备好）：
- `backend/zbpack.json`
- `frontend/zbpack.json`
- `backend/Dockerfile`
- `backend/src/main/resources/application-prod.yml`

**部署步骤**：
1. 打开 https://zeabur.com，GitHub 登录
2. New Project → 选 Hong Kong 区域
3. Deploy New Service → Git → 选 `my-recipes` 仓库 → 选 `backend` 目录
4. 再 Deploy New Service → Marketplace → MySQL
5. 在后端服务 Variables 里配置数据库连接信息：
   ```
   DATABASE_URL=jdbc:mysql://<MYSQL_HOST>:<MYSQL_PORT>/<MYSQL_DATABASE>?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
   DATABASE_USERNAME=<MYSQL_USERNAME>
   DATABASE_PASSWORD=<MYSQL_PASSWORD>
   JWT_SECRET=my-recipes-prod-secret-2024
   ```
6. 执行 `schema.sql` 初始化数据库
7. Vercel 部署前端：https://vercel.com → Import Git Repository → 选 `frontend` 目录
8. 在 Vercel 配置环境变量 `VITE_API_BASE_URL=https://<你的Railway后端域名>`

## 方案二：腾讯云/阿里云轻量服务器（最推荐长期使用）

**适合人群**：追求稳定、国内访问速度最快

| 服务 | 平台 | 费用 | 说明 |
|------|------|------|------|
| 前端 + 后端 + 数据库 | 腾讯云/阿里云轻量服务器 | ¥40-60/月 | 新用户首年特惠约 ¥74 |
| 学生机 | 阿里云/腾讯云 | ¥10/月 | 需学生认证 |

**优点**：
- 国内访问最快
- 完全自己掌控，无任何限制
- 一台服务器跑前端 + 后端 + MySQL

**购买链接**：
- 腾讯云轻量服务器：https://cloud.tencent.com/act/lighthouse
- 阿里云学生机：https://developer.aliyun.com/plan/student

**一键部署脚本**（准备好后可直接用）：
```bash
# 1. 安装 Docker 和 Docker Compose
curl -fsSL https://get.docker.com | sh

# 2. 克隆项目
git clone https://github.com/instanteternall/my-recipes.git
cd my-recipes

# 3. 启动全部服务
docker-compose up -d
```

需要准备的文件：`docker-compose.yml`（已规划，随时可生成）

## 方案三：完全免费组合（有限制）

**适合人群**：只是演示/学习用途，不在意速度和稳定性

| 服务 | 平台 | 费用 | 限制 |
|------|------|------|------|
| 前端 | [Vercel](https://vercel.com) | 免费 | 无限制 |
| 后端 | [Koyeb](https://koyeb.com) | 免费 | 512MB内存，无流量会休眠 |
| MySQL | [FreeSQLDatabase](https://freesqldatabase.com) | 免费 | 5MB存储，无需信用卡 |

**注意事项**：
- Koyeb 免费套餐无流量时会休眠，冷启动需要约 30 秒
- FreeSQLDatabase 存储空间很小（5MB），仅够测试
- 国内访问速度较慢（服务器在欧美）

**部署步骤**：
1. 注册 FreeSQLDatabase：https://freesqldatabase.com，获取 MySQL 连接信息
2. 用本地 MySQL 客户端连接，执行 `schema.sql`
3. 注册 Koyeb：https://app.koyeb.com，GitHub 登录
   - New App → Docker → GitHub 仓库 → `backend` 目录
   - 配置环境变量（同方案一的 Variables）
4. 注册 Vercel：https://vercel.com，GitHub 登录
   - New Project → Import `my-recipes` → Root Directory 选 `frontend`
   - 配置环境变量 `VITE_API_BASE_URL=https://<Koyeb后端域名>`

## 方案四：Docker Compose 本地/服务器一键部署

**适合人群**：已有服务器，或想在本地跑生产环境

只需一个 `docker-compose.yml` 文件即可启动前端 + 后端 + MySQL，随时可以生成。

**架构**：
```
Nginx（前端静态文件 + 反向代理）
    ↓
Spring Boot 后端（8080）
    ↓
MySQL 数据库（3306）
```
